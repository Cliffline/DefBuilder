This section collects the Steam Inventory Service material that DefBuilder needs in order to model item definitions accurately.

## Core Pages

- [overview.md](overview.md) explains the service model, trust boundaries, and implementation flow.
- [schema.md](schema.md) covers ItemDef structure, exact format rules, and the official schema example.
- [validation.md](validation.md) lists editor-facing validation rules and common failure cases.
- [official-sources.md](official-sources.md) maps each local page to the authoritative Steamworks page.

## Scope

These pages are a product reference for DefBuilder. They explain which Steam Inventory concepts the editor must understand, validate, and expose in the UI.

## Source Policy

- Narrative content is summarized from the official Steamworks documentation.
- Exact grammar, token formats, and official ItemDef examples are preserved where accuracy matters.
- Every page links back to the original source.
- Every Markdown page in this section carries its own file-specific `last updated` badge.
